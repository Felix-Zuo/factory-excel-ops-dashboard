产品定位

Product position

不是单一工厂模板，而是通用表格运营引擎

Not a single factory template, but a reusable spreadsheet operations engine

公开版本使用合成数据和通用运营术语，适合库存、需求、履约、补给、维修、项目交付、样品流转等表格密集型场景继续适配。

The public edition uses synthetic data and generic operations language, making it adaptable to inventory, demand, fulfillment, replenishment, maintenance, project delivery, sample flow, and other spreadsheet-heavy domains.

内容画像识别

Content profiling

通过表头、字段组合、样本值和弱文件名线索识别来源类型，降低文件改名和模板变化带来的风险。

Classifies sources through headers, field combinations, sample values, and weak filename hints.

统一字段契约

Unified field contract

把不同系统和人工表格里的项目、数量、日期、账户、伙伴等字段映射为标准结构。

Maps item, quantity, date, account, and partner fields into a stable standard model.

可配置指标

Configurable metrics

指标口径由 JSON 配置管理，避免把某个行业、品类或内部模板写死在代码里。

Metric definitions live in JSON config instead of being hard-coded for one domain.

智能体交接层

Agent handoff layer

将摘要转换为有边界的分析上下文，便于后续报告、检查建议和工作流自动化使用。

Converts summaries into bounded context for reports, review prompts, and workflow automation.

核心流程

Core workflow

从导出表格到可解释的运营看板

From exported spreadsheets to explainable operations dashboards

管线把输入文件、识别结果、标准记录、指标摘要和分析上下文分层保存，便于追踪问题，也便于替换私有适配器。

The pipeline keeps source files, classification results, normalized records, metric summaries, and analysis context in separate layers for traceability and adapter reuse.

  1. 01

    收集本地导出

    Collect local exports

    读取 CSV 或 Excel 导出文件，默认样例全部为合成数据。

    Reads CSV or Excel exports, with public samples kept fully synthetic.

  2. 02

    识别并标准化

    Classify and normalize

    按配置识别来源类型，并把噪声表头映射到通用字段。

    Classifies each source and maps noisy headers into common fields.

  3. 03

    计算指标

    Compute metrics

    生成库存、需求、履约、补给、产出等通用指标摘要。

    Builds generic summaries for inventory, demand, fulfillment, replenishment, and output.

  4. 04

    交付看板与上下文

    Deliver dashboard and context

    输出本地 HTML 看板、summary.json 和智能分析上下文。

    Exports a local HTML dashboard, summary.json, and agent-ready context.

能力边界

Capability surface

公开仓库保留可复用核心，把私有适配留在边界之外

The public repo keeps the reusable core and leaves private adapters outside

核心代码关注稳定的数据工作台能力：分类、映射、指标、看板、上下文和干净打包。真实系统凭证、内部模板和业务数据不进入公开仓库。

The core focuses on classification, mapping, metrics, dashboards, context, and clean packaging. Real credentials, internal templates, and business data stay out of the public repository.

配置验证

Config validation

在处理文件前检查来源画像、字段别名和指标规则，减少错误配置进入运行阶段。

Checks source profiles, field aliases, and metric rules before files are processed.

低置信度保护

Low-confidence guard

识别不确定时输出警告，不把未知文件静默当作正确来源处理。

Warns on uncertain classification instead of silently accepting unknown sources.

可审计摘要

Auditable summaries

JSON 摘要保存来源计数、指标卡片和运行警告，便于二次检查。

JSON summaries retain source counts, metric cards, and run warnings for review.

独立本地看板

Standalone local dashboard

HTML 输出不依赖远程服务，适合本地查看、归档和演示。

The HTML output does not depend on remote services and works for local review or demos.

Quality and safety

发布前检查

Release checks

  • 测试覆盖分类、字段映射、指标、验证和打包逻辑。
  • Tests cover classification, field mapping, metrics, validation, and packaging behavior.
  • CI 在 Python 3.10、3.11、3.12 上运行。
  • CI runs on Python 3.10, 3.11, and 3.12.
  • 打包检查阻止私有目录、生成输出、Excel 工作簿和嵌套归档进入发布包。
  • Package checks block private folders, generated output, Excel workbooks, and nested archives.

公开边界

Public boundary

仓库不包含真实 ERP、WMS、MES 导出，不包含客户、供应商、员工、订单、BOM、发运、财务或人事数据，也不包含私有工作簿模板、桌面二进制包和内部部署记录。

The repository contains no real ERP, WMS, or MES exports; no customer, supplier, employee, order, BOM, shipment, finance, or HR data; and no private workbook templates, desktop bundles, or internal deployment records.
