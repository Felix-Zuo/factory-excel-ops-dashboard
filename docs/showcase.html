Turn factory spreadsheets into an operational dashboard.

面向库存、订单、发货、采购和生产数据的本地化处理工具。先识别文件，再统一字段，最后生成可追溯的摘要和 HTML 看板。

核心能力

Core Capabilities

内容优先识别

Content-first classification

不依赖固定文件名，通过表头和内容样本判断库存、订单、发货、采购和生产数据。

Detects inventory, order, shipment, purchase, and production files from headers and sample values, not only file names.

标准字段映射

Standard field mapping

把不同来源的物料号、数量、日期、客户、供应商字段统一到同一数据模型。

Maps item, quantity, date, customer, and supplier fields from different exports into one model.

本地看板导出

Local dashboard export

输出独立 HTML 和 JSON 摘要，适合本地查看、汇报或继续接入其他工具。

Exports a standalone HTML dashboard and JSON summary for local review or downstream tools.

智能体接口

Agent interface

提供机器可读的接口契约，外部自动化工具可以读取能力、运行命令并打包自己的版本。

Provides a machine-readable contract so automation tools can inspect capabilities, run commands, and package their own edition.

接入方式

How to Integrate

团队可以复制配置文件，替换成自己的表格特征和字段别名。外部工具也可以读取接口文件，自动运行管道。

Teams can copy the config files and replace signatures and field aliases. External tools can read the interface file and run the pipeline directly.

python -m factory_excel_ops.cli run --input sample_data --output output python -m factory_excel_ops.cli agent-spec --output output/agent_interface.json
